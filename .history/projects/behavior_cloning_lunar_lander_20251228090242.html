Behavior Cloning for Lunar Lander

Abstract

This project explores behavior cloning for the LunarLander environment using a trained Deep Q-Network (DQN) as an expert policy. The expert DQN is rolled out to generate episodes, producing a golden dataset of state–action pairs. A multi-layer perceptron (MLP) student policy is then trained using supervised learning to directly map environment states to discrete control actions. The objective is to imitate expert behavior without reinforcement learning during deployment, resulting in a lightweight and efficient control policy.

Project Objectives

  • Generate expert demonstrations using a trained DQN agent.
  • Create a golden dataset of state–action pairs.
  • Train a supervised MLP policy to imitate expert behavior.
  • Evaluate the imitation performance of the cloned policy.

Technical Implementation

Expert Demonstrations

  • Rolled out a trained DQN expert in the LunarLander-v2 environment.
  • Recorded states and corresponding expert actions at every timestep.
  • Aggregated all demonstrations into a fixed golden dataset.
  • Split the dataset into training and validation subsets.

Student Policy

  • Multi-layer perceptron (MLP) trained via supervised learning.
  • Input: 8-dimensional LunarLander state vector.
  • Output: Discrete action representing thruster commands.
  • Trained using cross-entropy loss to match expert actions.

Training Workflow

  1. Run the DQN expert to generate complete episodes.
  2. Store state–action pairs as expert demonstrations.
  3. Train the MLP policy using supervised learning.
  4. Validate imitation accuracy on held-out expert data.
  5. Deploy the trained policy in the environment for evaluation.

Evaluation Highlights

Performance: Achieves rewards close to the DQN expert.

Stability: Consistent landings under standard conditions.

Efficiency: Fast inference with a compact MLP.

Lessons Learned

  • Behavior cloning can closely replicate strong expert policies.
  • Expert data quality directly impacts student performance.
  • Supervised policies are simple, fast, and easy to deploy.

Future Work

  • Extend behavior cloning using DAgger for improved robustness.
  • Apply the approach to continuous control environments.
  • Use the cloned policy as initialization for reinforcement learning.
