Value Iteration on Frozen Lake

Abstract

This project implements classical value iteration on the Frozen Lake problem from Gymnasium. Two complementary scripts showcase the algorithm in a lightweight headless configuration and an interactive pygame renderer. Both pipelines converge to an optimal value function, derive a greedy policy, and then evaluate the learned controller across a custom map configuration.

Project Components

The repository separates the learning logic from the visualization layer, allowing experiments in deterministic and stochastic settings:

Custom Frozen Lake Layout

  • Editable MY_MAP constant enables bespoke lake geometries with S, F, H, and G tiles.
  • Supports rectangular grids for rapid prototyping of difficulty variations.
  • Deterministic and slippery dynamics toggled through the Gymnasium environment flag.

Value Iteration Core

  • Bellman backups iterate until the value table change falls below a configurable tolerance.
  • Discount factor gamma tuned for convergence speed versus long-horizon planning.
  • Derived policy chooses greedy actions from the stabilized value function.

Visualization & Tooling

  • Headless mode prints policy symbols for quick inspection in the terminal.
  • Pygame-powered rendering animates the agent, reflecting deterministic or slippery dynamics.
  • Gymnasium toy-text APIs provide a compact interface for environment resets and transitions.

Policy Convergence Insights

Iterative evaluation reveals how value iteration systematically improves policy quality as the value table stabilizes:

Early Sweeps

  • Large Bellman residuals highlight uncertainty around hole-adjacent states.
  • Actions oscillate while the algorithm propagates goal rewards back through safe tiles.
  • Deterministic and slippery variants diverge in transitional value estimates.

Stabilized Policy

  • Value deltas shrink below tolerance, signalling convergence.
  • Greedy rollout reaches the goal reliably, even in stochastic transitions.
  • Updated policy tables become reusable baselines for further experiments.

Experiment Workflow

Configure Environment

1. (Optional) Activate a Python 3.9+ virtual environment.

2. Install dependencies: python -m pip install gymnasium[toy-text] numpy pygame.

3. Select deterministic or slippery transitions before initialization.

Compute Value Function

4. Launch value_iteration.py to run headless convergence.

5. Inspect value tables and policy arrows printed to the console.

6. Adjust gamma or tolerance to trade accuracy for speed.

Visualize Policy

7. Execute value_iteration_pygame.py for animated rollouts.

8. Toggle is_slippery to compare deterministic versus stochastic behavior.

9. Observe agent decisions as the greedy controller negotiates holes and goal tiles.

Future Directions

  • Scale to larger grids and adjust convergence criteria for performance.
  • Benchmark against policy iteration or asynchronous dynamic programming.
  • Integrate reward shaping to encourage specific traversal patterns.
  • Package the value iteration routine as a teaching notebook with interactive sliders.

Conclusion

The Frozen Lake project demonstrates how classical dynamic programming reliably yields optimal policies for discrete Markov decision processes. By pairing a concise implementation with optional visualization, the repository provides an accessible sandbox for studying convergence behavior, experimenting with map layouts, and contrasting deterministic and stochastic dynamics.
